# Тестирование API сервиса «Яндекс.Прилавок» (Postman)

## Описание проекта

API-тестирование сервиса доставки продуктов. Проверены ручки: наборы (kits), быстрая доставка (fast-delivery), корзина (orders) и другие.

**Какие проблемы решает проект?**  
API часто остаётся «чёрным ящиком»: фронтенд работает, а бэкенд возвращает ошибки. Я спроектировал и выполнил 75+ тестов, включая негативные сценарии и граничные значения, чтобы убедиться, что API корректно обрабатывает любые входные данные. Без таких проверок критический баг (например, 500 вместо 400) может уйти в прод и разрушить пользовательский опыт. Проект показывает, что я умею находить дефекты на ранней стадии и документировать их так, чтобы разработчики могли быстро исправить проблему.

**Почему я его создавал?**  
Чтобы доказать, что я понимаю, как работает API: методы, статус-коды, форматы JSON/XML, граничные значения. Я умею проектировать проверки системно, а не выполнять их хаотично.

**Стек (инструменты):**  
- Postman (ручное выполнение запросов)  
- JSON, XML  
- Техники тест-дизайна (граничные значения, классы эквивалентности)
